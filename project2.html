Multi-Cloud Kubernetes Federation

Project Overview

To avoid vendor lock-in and enhance disaster recovery capabilities, the organization decided to pursue a multi-cloud strategy. I was tasked with leading the technical design and implementation of a federated Kubernetes platform spanning both AWS (EKS) and Google Cloud (GKE).

Key Responsibilities:

  • Evaluated and selected Google Anthos as the central control plane for managing clusters across different cloud providers.
  • Architected a global service mesh using Istio to manage cross-cluster traffic, enforce security policies, and provide unified telemetry.
  • Utilized Terraform to provision and manage the underlying infrastructure for the Kubernetes clusters in both clouds, ensuring consistency.
  • Developed a GitOps-based deployment strategy using ArgoCD to synchronize application and configuration state across all federated clusters.

Results & Achievements:

The successful implementation of the federated platform provided a single pane of glass for managing our entire container ecosystem. We achieved active-active disaster recovery, allowing for seamless failover between cloud providers with no data loss. This strategic initiative also gave the business leverage to negotiate better pricing with cloud vendors.
